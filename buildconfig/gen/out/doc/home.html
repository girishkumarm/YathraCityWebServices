The InBlox API

Version: 0.1

Last Updated: 8 Feb 2016 12:39:30 GMT

Introduction

This guide specifies a RESTful API for creating and managing guest identity verification and background checks, reservation payment and escrow services, damage waiver and insurance sales, post-reservation evaluation of the host and guest, home profile management, calendar updates, and account creation.

Use of the API is via the HTTP protocol. The GET, POST, PUT, and DELETE requests are all used, with standard interpretations for the protocol - POST usually creates, PUT updates, DELETE deletes, and GET reads a record or records. Construct representations are in JSON.

The API presupposes no particular structure in the URI space. The starting point is a URI, supplied by InBlox. The specification of the InBlox API includes:

  • Common behaviors that apply across all requests and responses.
  • Construct models, which describe the JSON data structures used in requests and responses.
  • The requests that may be sent to InBlox, and the expected responses.

This API operates on the following primary resource types:

  • BookedCarDetails:input to register car details
  • RegisterCarInput:input to register car details
  • FetchCarDetails:for fetching the car details
  • ListOfAvailableCars:AvailableCars
  • CheckAvailabilityInput:input to register car details
  • TotalNumberOfUsers:car avaliable on that particular day
  • Message:car avaliable on that particular day
  • CouponDetails:car avaliable on that particular day
  • ListOfCouponsUsed:LoginResponse which describes the user is real or not
  • ConfirmBookedCarDetails:input to register car details
  • DriverDetails:input pojo which describes all the required information of the driver
  • ConfirmDriverAvailability:input pojo which describes all the required information of the driver to confirm his availability
  • ResponseMessage:ResponseMessage for acknowledging the services
  • LogInput:input to log
  • LogInputList:input to log
  • RequestObject:request fields
  • LoginInput:input pojo which describes all the required information of the user
  • LoginResponse:LoginResponse which describes the user is real or not
  • RegisterUser:input pojo which describes all the required information of the user
  • RegisterUserResponse:RegisterUserResponse
